What this is

Real-time voice pipeline

Wake word → STT → LLM → TTS, all streaming in parallel. Target <500ms perceived latency from end of speech to first audio output.

Persistent memory

Vector + graph memory so Jarvis remembers your preferences, past tasks, people, and habits across sessions — like a real assistant.

Agentic tool use

Execute code, browse the web, control your OS, manage calendar/email, query APIs — not just answer questions, but take actions.

Multimodal vision

Screen-aware: Jarvis can see your display, analyze images you point a camera at, and understand context from what you're looking at.

Proactive intelligence

Background monitoring: weather, meetings, news, system health — Jarvis surfaces what matters before you ask, like a true Iron Man HUD.

Local + cloud hybrid

Wake word and STT run on-device for privacy and zero latency. Heavy reasoning routes to cloud or local GPU depending on task complexity.

Core philosophy

Tony Stark's Jarvis was powerful not because it answered questions — but because it anticipated needs, executed tasks, and knew Tony's context deeply. That requires three things modern LLM assistants usually miss: ultra-low latency (under 500ms), persistent memory across sessions, and genuine agentic tool use. This blueprint builds all three.

Voice pipeline

Wake word Porcupine (Picovoice) — on-device, 10ms latency, custom "Hey Jarvis" model On-device
Speech-to-text Deepgram Nova-3 streaming — <150ms TTFT, 36 languages, noise-robust Cloud API
STT fallback Whisper.cpp (local) — offline mode, privacy-critical tasks On-device
Text-to-speech ElevenLabs Flash v2.5 — 75ms latency, custom Jarvis voice clone Cloud API
TTS fallback Kokoro-82M (local) — open source, 95% quality at zero cost On-device
VAD / end-of-turn Deepgram built-in end-of-turn detection (~260ms) + Silero VAD locally Hybrid

Brain (LLM layer)

Primary LLM Claude Sonnet 4 (claude-sonnet-4-6) — reasoning, tool use, long context API
Fast LLM Claude Haiku 4.5 — <200ms, for quick lookups and routing decisions API
Local LLM Qwen2.5-7B-Instruct via Ollama — private / offline tasks, no API cost On-device
Vision Claude Sonnet 4 vision + LLaVA local — screen capture, camera feed analysis Hybrid
Embeddings text-embedding-3-small (OpenAI) or Nomic Embed local Hybrid

16-week build plan

Phase 1 · Weeks 1–2
Voice core — the talking skeleton
Wake word with Porcupine. Deepgram streaming STT. FastAPI WebSocket server. ElevenLabs TTS with custom Jarvis voice. Basic Claude Haiku integration. End-to-end voice loop that listens, thinks, and speaks. Target: under 800ms round-trip.
Phase 2 · Weeks 3–5
Memory — Jarvis remembers you
Chroma vector DB for semantic memory. SQLite for structured preferences and history. Memory injection into every prompt. Implement memory consolidation: after each session, summarize and store key facts. Graph memory for people and relationships using Neo4j or Kuzu.
Phase 3 · Weeks 6–9
Tools — Jarvis acts, not just talks
Web search via Playwright. Calendar read/write (Google Calendar MCP). Email via Gmail MCP. OS control: open apps, read screen, take screenshots. Python code runner (sandboxed). Smart home via Home Assistant API. Weather, news, finance APIs. Intent-to-tool router using Haiku as a fast classifier.
Phase 4 · Weeks 10–13
Vision + intelligence — Jarvis sees
Screen capture pipeline — Jarvis sees what you're working on and can comment without being asked. Camera integration for real-world object and person recognition. Vision-grounded reasoning with Claude Sonnet 4 vision. Proactive alerts: meeting reminders, weather changes, system health, price triggers — surface via ambient HUD.
Phase 5 · Weeks 14–16
HUD + polish — the Stark experience
React + Three.js holographic HUD with animated arc reactor visualizer, real-time waveform, task status cards. Emotion/tone detection in voice. Personalized wake phrase training. Local LLM fallback (Qwen2.5) for offline/private mode. Latency optimization: sentence-level streaming, caching, prefetch. Security hardening.

What Jarvis can do

Knowledge & search

Web search via Playwright, news feeds, Wikipedia, Wolfram Alpha, real-time data via REST APIs. Synthesizes results before speaking.

Calendar & scheduling

"Block 2 hours for deep work tomorrow morning" — reads, creates, and moves events via Google Calendar MCP.

Email management

Summarize inbox, draft and send emails, flag priority items, search by topic — via Gmail MCP with Claude drafting.

OS control

Open apps, switch windows, read screen text, automate workflows, trigger keyboard shortcuts — via PyAutoGUI and system APIs.

Code execution

Run Python sandboxed, compute, process data, generate and test scripts — secure subprocess runner with output piped back to LLM.

Smart home

"Dim the lights and set thermostat to 22" — Home Assistant REST API integration with device discovery.

Music & media

Spotify playback control, YouTube queue, podcast search — via Spotify Web API and system media commands.

Finance & stocks

Portfolio check, price alerts, news on holdings, crypto prices — via Alpha Vantage and CoinGecko APIs.

Vision & screen

"What's on my screen?" or "Who is this person?" — screenshot analysis and camera feed via Claude Sonnet vision.

Reminders & tasks

Natural language task creation, smart follow-ups — "remind me when Abhay replies", stored in SQLite with proactive delivery.

Travel & logistics

Flight status, directions, restaurant search, packing lists, hotel options — via Google Maps API and Amadeus travel API.

Health & habits

Hydration reminders, Pomodoro timer, posture alerts, sleep schedule — proactive background monitoring with gentle nudges.
